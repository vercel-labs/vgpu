# StructuredUniform

`StructuredUniform` is a schema-typed uniform buffer for custom WebGPU pipelines.
It uses the same string-literal uniform schema dialect as `material()` and the
same WGSL alignment machinery, but it does not create a render pipeline or force a
fixed bind group shape. Use it when you need a generated uniform-buffer ABI while
still composing your own bind groups, blend state, render targets, or shader
entry points.

## Constructor

```ts
const params = new StructuredUniform(device, {
  label: "scene-params",
  schema: {
    viewProj: "mat4x4f",
    cameraPos: "vec3f",
    time: "f32",
  },
  visibility: ["vertex", "fragment"], // optional; default is vertex + fragment
});
```

- `schema`: non-empty record of field names to `WgslUniformType` values. Field
  insertion order is the WGSL struct member order.
- `label`: optional debug label for the backing buffer and the lazy bind group
  objects.
- `visibility`: shader stages for the lazy binding-0 layout. Pass
  `GPUShaderStageFlags`, a string, or an array such as `["compute"]`.

Supported field types are the same flat uniform types supported by `material()`:
`f32`, `u32`, `i32`, `vec2f`, `vec3f`, `vec4f`, `vec2u`, `vec3u`, `vec4u`,
`vec2i`, `vec3i`, `vec4i`, `mat3x3f`, and `mat4x4f`. Nested structs and arrays
are intentionally out of scope for v1.

## Members

- `structured.schema`: the original schema object.
- `structured.layout`: `UniformLayoutInfo` with fields, offsets, and byte size.
  This is exposed so future layout validation can consume the generated ABI.
- `structured.byteSize`: the derived uniform struct byte size.
- `structured.offsets`: byte offsets keyed by schema field name.
- `structured.buffer`: the VGPU `Buffer` wrapper, created with
  `uniform | copy_dst` usage.
- `structured.gpu`: the underlying `GPUBuffer`, so `bind.resource(0, structured)`
  works anywhere a VGPU buffer-like object is accepted.
- `structured.bindGroupLayout` / `structured.bindGroup`: lazy binding-0 uniform
  layout and bind group for the common one-buffer case.
- `structured.destroy()` / `structured.dispose()` / `structured[Symbol.dispose]()`:
  release the backing buffer once; idempotent.

## Partial writes

`write()` accepts a partial object and merges it into a zero-initialized CPU
scratch buffer. Missing keys keep their previous values. Every call uploads the
whole uniform buffer in one `queue.writeBuffer` call.

```ts
params.write({ viewProj, cameraPos: eye });
params.write({ time: performance.now() / 1000 }); // viewProj/cameraPos persist
```

Scalars must be numbers. Vector and matrix fields accept arrays, `Float32Array`,
`Uint32Array`, or `Int32Array`; `write()` checks the exact runtime length. For
example, `vec3f` must receive
exactly three values and `mat3x3f` must receive exactly nine values. `mat3x3f`
values are expanded into WGSL's 16-byte column stride, so the padding cannot drift
from the generated layout.

Unknown keys throw `VGPU-CORE-INVALID-USAGE`, which catches field-name typos at
first write.

## WGSL generation

`wgsl(structName = "Uniforms")` emits only the struct declaration. Add your own
`@group` / `@binding` line to match the bind group that consumes the buffer.

```ts
const shader = `${params.wgsl("Params")}
@group(0) @binding(0) var<uniform> params: Params;
${body}`;
```

Example output:

```wgsl
struct Params {
  viewProj: mat4x4<f32>,
  cameraPos: vec3<f32>,
  time: f32,
};
```

## Composing into custom bind groups

Use the lazy bind group for the simple one-buffer case:

```ts
pass.setBindGroup(0, params.bindGroup);
```

Or compose the buffer into a larger bind group:

```ts
const group = createBindGroup(device, {
  layout,
  entries: [
    bind.resource(0, params),
    bind.resource(1, texture),
    bind.resource(2, sampler),
  ],
});
```

## Group by update frequency

Group uniforms by **update frequency**, not by conceptual domain. Use one
`StructuredUniform` per update tier:

| Tier | Contents | Write cadence |
| --- | --- | --- |
| Per-frame | camera, time, DPR, global lighting | once per frame |
| Per-pass / per-draw | pass kind, blend params | once per pass or draw |
| Rarely-changing | material constants, grid config | on interaction only |

`queue.writeBuffer` has fixed per-call overhead, each `setBindGroup` is a state
change, and default WebGPU limits (`maxUniformBuffersPerShaderStage = 12`,
`maxBindGroups = 4`) make many tiny uniform buffers expensive. A single
mega-uniform is the opposite trap: changing one per-pass field can force repeated
mid-frame rewrites or duplication. One struct per tier avoids both extremes.

## Relationship to `material()` and `Uniform`

`material()` is still the quickest path when its fixed pipeline shape works for
your renderer. `StructuredUniform` is for custom pipelines that still want the
same schema-to-layout safety.

`Uniform` is a raw byte buffer helper: you provide the byte size and packed data.
`StructuredUniform` derives byte size and offsets from a schema and writes fields
by name.

## Not in v1

`StructuredUniform.slots()` is described as a v1.1 follow-up in the original
proposal and is not included in v1. Use `UniformPool` for streaming per-draw ring
allocation today, or create multiple `StructuredUniform` instances when you need
stable per-pass values before the slots helper lands.
