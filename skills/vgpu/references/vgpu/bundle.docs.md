# `gpu.bundle` and render-bundle replay

Bundles are explicit. Record static commands once, replay them inside a frame pass, and keep dynamic draws outside the bundle.

```ts
const scene = gpu.target({ format: "rgba16float", depth: true });
const staticScene = gpu.bundle({ target: scene }, (b) => {
  b.draw(floor);
  b.draw(walls);
});

gpu.frame.loop((f) => {
  f.pass({ target: scene }, (p) => {
    p.bundles(staticScene);
    p.draw(player);
  });
});
```

R3 staleness: a bundle freezes commands and bind groups, not buffer contents. `draw.set({ fog: 0.2 })` is safe because JS values write in-place. `draw.set({ tex: otherTexture })` changes resource identity and stales the bundle; replay detects this in dev and tells you to re-record.

Ping-pong with bundles uses two recorded bundles, one for each stable resource identity:

```ts
const even = gpu.bundle({ target: ping.write }, (b) => { sim.set({ src: ping.read }); b.draw(sim); });
ping.swap();
const odd = gpu.bundle({ target: ping.write }, (b) => { sim.set({ src: ping.read }); b.draw(sim); });
ping.swap();

gpu.frame.loop((f) => {
  f.pass({ target: ping.write }, (p) => p.bundles(gpu.frameCount % 2 ? odd : even));
  ping.swap();
});
```

## Ownership and performance defaults

- WGSL is the source of truth. Declare every `@group/@binding` in the shader and bind by name with `set()`.
- JS values passed to `set()` are lib-owned and are written in-place (R1/R2), so animated uniforms do not recreate bind groups.
- Resources (`Uniform`, storage buffers, textures, targets, samplers, claimed bind groups) are user-owned; vgpu only binds their identity.
- Time is explicit JS (`gpu.time`, `gpu.deltaTime`, `gpu.frameCount`). Resolution lives on targets (`target.size`, `target.texelSize`).
